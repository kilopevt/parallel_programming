# Лабораторная работа №1  
## Умножение квадратных матриц: анализ времени выполнения  

В данной работе реализовано умножение двух квадратных матриц с замерами времени выполнения для различных размерностей.  
Используемые размеры матриц: **200, 400, 800, 1200, 1600, 2000**.

---

## Состав проекта

- **`gen.py`** – генерирует квадратные матрицы указанных размеров, заполняя их случайными целыми числами от 0 до 1000. Матрицы сохраняются в папку `matrices/` в формате CSV с именами `A_<размер>.csv` и `B_<размер>.csv`.
- **`main.cpp`** – основная программа. Выполняет умножение двух матриц, замеряет время выполнения (в миллисекундах) и дописывает результат в указанный CSV-файл. Поддерживает интерактивный ввод путей и многократные запуски.
- **`result.py`** – считывает CSV-файл с результатами, строит график зависимости времени выполнения от размера матриц и сохраняет его как `time_vs_result.png`.

---

## Порядок выполнения работы

1. **Генерация матриц**  
   Запустите скрипт `gen.py`.  
   В папке `matrices/` будут созданы файлы:  
   `A_200.csv`, `B_200.csv`, `A_400.csv`, `B_400.csv`, … `A_2000.csv`, `B_2000.csv`.

2. **Умножение матриц**  
   Запустите `main.cpp` (предварительно скомпилировав его, например, `g++ -O2 -o main.exe main.cpp`).  
   Программа запросит:
   - путь к файлу первой матрицы (A);
   - путь к файлу второй матрицы (B);
   - путь к файлу для сохранения результатов (например, `results.csv`).  
   После вычисления будет выведено время умножения. Программа предложит выполнить ещё одно умножение (введите `y` для продолжения или `n` для завершения).  
   Результаты дописываются в указанный CSV-файл. Для каждого размера рекомендуется запускать умножение отдельно, чтобы в итоговом файле оказались все шесть строк.

3. **Построение графика**  
   В файле `result.py` укажите путь к CSV-файлу с результатами (если он не совпадает с именем по умолчанию).  
   Запустите скрипт: `python result.py`.  
   Будет построен график и сохранён в файл `time_vs_result.png`.

## Вывод

Временная сложность использованного алгоритма умножения матриц (классический тройной вложенный цикл) составляет **O(N³)**.  
Полученные экспериментальные данные подтверждают эту теоретическую оценку: при увеличении размера матрицы в 2 раза время выполнения возрастает примерно в 8 раз (для больших размеров отклонения обусловлены особенностями кэш-памяти и фоновыми процессами ОС).
